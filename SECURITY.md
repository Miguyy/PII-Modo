# Security Policy - <img src="./apps/modo_front-end/Modo/src/images/modoIcon.png" alt="Modo Icon" width="130" height="30"/>

Although this repository is primarily educational, we take security and privacy seriously. This document explains how to report security vulnerabilities or sensitive disclosures related to the Modo project.

## Supported Versions

We maintain the latest `main` branch. Historical tags or archived lab versions are provided as-is and may not receive security updates.

## Reporting a Vulnerability

Preferred reporting channels (in order):

1. Create a private GitHub Security Advisory for this repository (recommended), or
2. Open a private GitHub issue and label it `security`, or
3. Contact one of the project maintainers via their GitHub profiles listed in the README: Miguel Machado (https://github.com/Miguyy), Manuel Teixeira (https://github.com/Manutex78), or Linda Silva (https://github.com/LindaGlahy).

When reporting, please include:

- A clear description of the issue and potential impact.
- Steps to reproduce or a proof of concept (screenshots, scripts, data samples).
- Any mitigating factors, suggested fixes, or patches if available.

Allow at least 5 business days for an initial response before publicly disclosing the issue.

Please do **not** open a public GitHub issue for security reports or leaks of sensitive data (student data, grades, credentials, or secrets).

## Scope

- Repository code, scripts, and datasets.
- CI workflows and automation.
- Documentation or lab instructions that might expose secrets or sensitive infrastructure details.

Out of scope:

- Third-party services not controlled by the maintainers (e.g., personal cloud accounts, unmanaged third-party services).
- Intentionally insecure lab environments used for pedagogical exercises.

## Coordinated Disclosure

We appreciate coordinated disclosure. After verifying a reported issue, maintainers may request additional time to develop and deploy a fix before public disclosure. Credit may be given in release notes unless anonymity is requested.

Thank you for helping keep the Modo project secure and the community safe.
